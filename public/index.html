LiverTracker clinical scores API

REST JSON API for MELD, MELD-Na, MELD 3.0, Child-Pugh, liver enzyme checks, and FibroScan interpretation — same engine as the open-source monorepo.

Base URL Endpoints curl examples Responses How to cite GitHub

Base URL

All versioned routes are under /api/v1. On this deployment, the origin you are viewing is the API host (static docs + serverless routes).

Replace $BASE in the examples below with your host, for example this site’s origin, or a custom domain you attach in Vercel. 

export BASE="https://meld-child-pugh.vercel.app"   # example; use your production URL

Endpoints

Send Content-Type: application/json on POST bodies.

Method Path Description
GET /api/v1/health Liveness; includes engine + citation metadata in JSON
GET /api/v1/scoring-systems Catalog / metadata for client apps
POST /api/v1/calculate Run all applicable tools from one JSON object (optional fields)
POST /api/v1/meld MELD only
POST /api/v1/meld-na MELD-Na
POST /api/v1/meld3 MELD 3.0
POST /api/v1/child-pugh Child-Pugh
POST /api/v1/liver-enzymes Liver enzyme checker (at least one analyte)
POST /api/v1/fibroscan Liver stiffness (kPa); optional CAP (dB/m)
Disclaimer Educational and clinical decision support only. Not a substitute for professional judgment or institutional protocols. CORS is open for developer convenience; add your own policies if you expose sensitive workloads.

curl examples

Paste into a terminal after setting BASE.

Shell line continuation Put \ at the end of the line with no characters after it (a trailing space after \ breaks the continuation, so curl may omit -H / -d).
If jq fails with a parse error The body may not be JSON — for example a Vercel error or timeout page. Run the same command without | jq . to inspect the raw response.

Health check

curl -sS "$BASE/api/v1/health" | jq .

Scoring systems catalog

curl -sS "$BASE/api/v1/scoring-systems" | jq .

MELD

curl -sS -X POST "$BASE/api/v1/meld" \
  -H "Content-Type: application/json" \
  -d '{"bilirubin":2.0,"creatinine":1.2,"inr":1.4,"onDialysis":false}' | jq .

MELD-Na

curl -sS -X POST "$BASE/api/v1/meld-na" \
  -H "Content-Type: application/json" \
  -d '{"bilirubin":2.0,"creatinine":1.2,"inr":1.4,"sodium":132,"onDialysis":false}' | jq .

MELD 3.0

curl -sS -X POST "$BASE/api/v1/meld3" \
  -H "Content-Type: application/json" \
  -d '{"bilirubin":2.0,"creatinine":1.2,"inr":1.4,"sodium":132,"albumin":3.2,"sex":"female","onDialysis":false}' | jq .

Child-Pugh

ascites: none | mild | moderate_severe — encephalopathy: none | grade_1_2 | grade_3_4 

curl -sS -X POST "$BASE/api/v1/child-pugh" \
  -H "Content-Type: application/json" \
  -d '{"bilirubin":2.0,"albumin":3.0,"inr":1.5,"ascites":"mild","encephalopathy":"none"}' | jq .

Liver enzymes

At least one of alt, ast, ggt, alp, bilirubin.

curl -sS -X POST "$BASE/api/v1/liver-enzymes" \
  -H "Content-Type: application/json" \
  -d '{"alt":120,"ast":90,"alp":200,"ggt":65,"bilirubin":1.0}' | jq .

FibroScan interpreter

curl -sS -X POST "$BASE/api/v1/fibroscan" \
  -H "Content-Type: application/json" \
  -d '{"liverStiffness":10.2,"capScore":280}' | jq .

Universal /calculate

One payload: the server runs every tool for which required inputs are present (may return null for tools that lack data). 

curl -sS -X POST "$BASE/api/v1/calculate" \
  -H "Content-Type: application/json" \
  -d '{
    "bilirubin": 2.0,
    "creatinine": 1.2,
    "inr": 1.4,
    "sodium": 132,
    "albumin": 3.2,
    "sex": "female",
    "onDialysis": false,
    "ascites": "none",
    "encephalopathy": "none",
    "alt": 85,
    "liverStiffness": 8.5
  }' | jq .

Response shape

Successful responses are JSON objects that include a meta object (apiVersion, calculationEngine, timestamp, disclaimer, and citation with repository URL and OSF tool registrations). Tool results live in addition to meta (e.g. result, or multiple keys for /calculate).

Errors return HTTP 4xx/5xx with meta and an error object:

{
  "meta": { ... },
  "error": {
    "code": "INVALID_INPUT",
    "message": "Request body must be JSON"
  }
}

How to cite

Cite the software / API host, the GitHub repository, and — when your work concerns a specific tool — the matching Open Science Framework (OSF) registration DOI. Full guidance: docs/CITATION.md.

GitHub “Cite this repository”

The repo ships CITATION.cff for GitHub’s citation export (APA, BibTeX, etc.): github.com/priyamjyotsna/MELD-CHILD-PUGH.

Example BibTeX (monorepo)

@software{meld_child_pugh_2026,
  author    = {Priyam, Jyotsna},
  title     = {MELD-CHILD-PUGH: Liver clinical scores and LiverTracker tools},
  year      = {2026},
  url       = {https://github.com/priyamjyotsna/MELD-CHILD-PUGH},
  license   = {MIT}
}

When you use this HTTP API in methods, record the API base URL, deployment date or commit, and pin the calculationEngine string from /api/v1/health if you need reproducibility.
